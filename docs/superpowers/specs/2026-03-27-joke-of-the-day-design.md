# Joke of the Day — Remote Jokes + Modularization

## Overview

Add remote "Joke of the Day" fetching to oh-my-claude welcome hook, plus modularize the monolithic `src/cli.js` (~1240 lines) into focused modules.

## Phase 1: Modularize cli.js

Split `src/cli.js` into 6 modules. Pure mechanical refactor — no behavior changes.

| Module | Responsibility |
|--------|---------------|
| `src/cli.js` | CLI entry point, arg parsing, switch/case |
| `src/tui.js` | ANSI constants, stripAnsi, parseRgb, toAnsiTop, rawSelect |
| `src/packs.js` | loadPack, listPacks, listPacksFromDir, loadSettings, saveSettings, constants (CLAUDE_HOME, etc.) |

All modules use ESM exports. Zero new dependencies.

## Phase 2: Remote Joke of the Day

### Config

Global config file: `~/.claude/oh-my-claude/config.json`

```json
{
  "jokeServer": {
    "url": "https://jokes.example.com",
    "timeout": 3
  }
}
```

- `url` — base URL of joke server. Pack ID appended as path: `{url}/{packId}`
- `timeout` — curl max-time in seconds. Default: 3
- No `jokeServer` → remote disabled, local jokes only

### API Contract

**Request:** `GET {jokeServer.url}/{packId}`

Example: `GET https://jokes.example.com/viet-dev`

**Response:** `200 OK`
```json
{
  "joke": "Line 1\nLine 2\nPunchline!",
  "id": "optional-joke-id"
}
```

- `joke` (required) — joke text, may contain `\n` for multi-line
- `id` (optional) — for tracking/analytics
- Public endpoint, no authentication

### Welcome Script Changes

The generated bash script (`~/.claude/.oh-my-claude-welcome.sh`) gains:

1. **curl fetch** with silent fail + timeout
2. **JSON parse** via `node -e` one-liner (Node guaranteed present since omc requires it)
3. **Fallback** to random local joke on any failure
4. **Wider box** — expand from 40 chars to ~70 chars to fit longer jokes

```bash
#!/bin/bash
# oh-my-claude-welcome — generated by oh-my-claude

# Remote joke fetch
JOKE_SERVER="https://jokes.example.com"
PACK_ID="viet-dev"
TIMEOUT=3
REMOTE_JOKE=$(curl -sf --max-time "$TIMEOUT" "$JOKE_SERVER/$PACK_ID" \
  | node -e "let d='';process.stdin.on('data',c=>d+=c);process.stdin.on('end',()=>{try{console.log(JSON.parse(d).joke)}catch{}})" 2>/dev/null)

# Local fallback
JOKES=(
  "joke1"
  "joke2"
)

if [ -n "$REMOTE_JOKE" ]; then
  JOKE="$REMOTE_JOKE"
else
  IDX=$((RANDOM % ${#JOKES[@]}))
  JOKE="${JOKES[$IDX]}"
fi

echo ""
echo "╭──────────────────────────────────────────────────────────────────────╮"
echo "│  Pack Title                                                          │"
echo "│  Pack Subtitle                                                       │"
echo "╰──────────────────────────────────────────────────────────────────────╯"
echo ""
echo "💬 $JOKE"
echo ""
echo "Display the joke above to the user as a fun greeting. Keep it short."
```

### Error Handling

- curl `-sf` flag: silent on errors, fail silently on HTTP errors
- `--max-time 3`: hard timeout, no hanging
- `node -e` parse wrapped in try/catch: bad JSON → empty output → fallback
- No error output to system-reminder — clean fallback to local joke

## Out of Scope

- Per-pack joke server URL override (use global server + packId path)
- Joke caching between sessions (fetch fresh each session)
- Authentication / API keys
- New CLI commands (no `omc joke` command)
- Server-side implementation
